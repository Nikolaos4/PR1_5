# 🧪 Практические работы по веб-разработке

Данный репозиторий содержит выполненные задания по курсу разработки веб-приложений. Каждая папка соответствует отдельной практической работе и демонстрирует изученные технологии и подходы.

## 📁 Структура

| Папка | Тема |


## 🧪 Практическая работа №2 – REST API на Express

**Цель:** создать сервер на Node.js с использованием Express, реализующий CRUD для сущности «товар».

**Что сделано:**
- Написан сервер (`app.js`) с маршрутами:
  - `GET /products` – получить все товары
  - `GET /products/:id` – получить товар по id
  - `POST /products` – добавить новый товар 
  - `PATCH /products/:id` – обновить товар
  - `DELETE /products/:id` – удалить товар
- Товар содержит поля: `id`, `name`, `price`.
- Данные хранятся в оперативной памяти (массив).

**Технологии:** `express`

**Как запустить:**
```bash
cd pr2
npm install express
node app.js
```
Сервер будет доступен по адресу `http://localhost:3000`.

## 🧪 Практическая работа №4 – Интернет-магазин (React + Express)

**Цель:** разработать полноценное веб-приложение с клиентской частью на React и серверной на Express, связав их через REST API.

### 🖥️ Сервер (папка `pr4/backend`)

- Реализован сервер на Express с CRUD для товаров (поля: `id`, `name`, `category`, `description`, `price`, `stock`).
- Использован `nanoid` для генерации уникальных идентификаторов.
- Настроен CORS для запросов с фронтенда (`http://localhost:3001`).
- Добавлено 10 начальных товаров.

**Технологии:** `express`, `cors`, `nanoid`

### 🎨 Клиент (папка `pr4/frontend`)

- Создан с помощью `create-react-app`.
- Компонентная архитектура:
  - `ProductsPage` – главная страница со списком товаров и модальным окном.
  - `ProductsList` – отображение списка.
  - `ProductItem` – карточка товара.
  - `ProductModal` – форма создания/редактирования.
- Для стилей используется **Sass** (переменные, миксины, вложенность).
- HTTP-запросы выполняются через **axios** (файл `api/index.js`).

**Технологии:** `react`, `sass`, `axios`, `cross-env`

### 🚀 Запуск проекта

**Бэкенд:**
```bash
cd pr4/backend
npm install
node app.js
```
Сервер запустится на `http://localhost:3000`.

**Фронтенд:**
```bash
cd pr4/frontend
npm install
npm start
```
Приложение откроется на `http://localhost:3001`.

## 🧪 Практическая работа №5 – Документирование API с помощью Swagger

**Цель:** добавить к уже существующему бэкенду (из практики №4) автоматическую генерацию OpenAPI-спецификации и интерактивный интерфейс Swagger UI.

### 🔧 Что сделано

- Установлены пакеты: `swagger-jsdoc`, `swagger-ui-express`.
- В файл `app.js` добавлена конфигурация Swagger:
  - Описание API (название, версия, сервер).
  - Подключение Swagger UI по маршруту `/api-docs`.
- Описана схема `Product` (все поля, обязательные, пример).
- К каждому эндпоинту (`GET`, `POST`, `GET/:id`, `PATCH/:id`, `DELETE`) добавлены JSDoc-комментарии с указанием:
  - краткого описания,
  - тегов для группировки,
  - параметров пути (где нужно),
  - ожидаемого тела запроса,
  - возможных ответов и ссылок на схему.

### 📖 Результат

- Документация доступна по адресу `http://localhost:3000/api-docs`.
- Интерфейс позволяет не только читать описание, но и отправлять реальные запросы к API (кнопка **Try it out**).
- Фронтенд из практики №4 продолжает работать без изменений.

### 🚀 Запуск (аналогично практике №4)

```bash
cd pr5/backend
npm install
node app.js
```
Затем открыть `http://localhost:3000/api-docs` в браузере.
